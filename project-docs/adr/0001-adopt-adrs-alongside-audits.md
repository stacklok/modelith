# Adopt ADRs alongside audits, both under `project-docs/`

We keep two kinds of record and had only one. `audits/` captures retrospective
multi-agent review snapshots; it does not capture a decision at the moment it
is made, so intentions and plans drifted out of the tree between audits. We add
`adr/` for forward-looking decision records: an ADR states the direction chosen
and why, as it is chosen.

Both live under `project-docs/` to keep the repository root clean and to group
the durable project records in one browsable place. ADRs go in
`project-docs/adr/`, not `docs/adr/`, because `docs/` is the published
Docusaurus site (its sidebar is autogenerated from the whole directory); a
decision record does not belong on modelith.sh, and the top-level `project-docs/`
location keeps it off the site without an underscore-prefix workaround.

The `.scratch/` directory (repo-local, gitignored) holds throwaway working
files so they stay browsable in the checkout instead of scattering into `/tmp`.
It is deliberately not under `project-docs/`, which is for durable records.
